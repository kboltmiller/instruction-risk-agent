# Changelog

All notable changes to this project will be documented in this file.

This project follows semantic versioning in spirit, though early versions may evolve rapidly.

---

## [0.2.0] – 2026-01-14

### Added
- Mitigation considerations surfaced alongside identified risks
- Explicit separation between identified risks, missing safeguards, and mitigation signals
- Documentation updates clarifying non-prescriptive intent

### Changed
- API response schema expanded to include `mitigation_considerations`
- README updated to reflect current behavior and positioning

### Notes
This release emphasizes clarity and explainability without altering or rewriting user instructions.

---

## [0.1.0] – Initial Pilot Release

### Added
- FastAPI-based evaluation endpoint
- Heuristic-based instruction risk detection
- Risk level classification (low / medium / high)
- Human-readable explanations for identified risks

### Notes
Initial experimental release focused on feasibility and core signal quality.
